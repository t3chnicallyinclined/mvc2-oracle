One-click bake (Chrome / Edge — no install, no server)
  1. Pick a character. Recolor palette swatches, and/or pick an animation → click a part → paint.
  2. Hit ⬇ bake to ROM → pick your GDI folder (the one with track03.bin) → confirm.
  3. The bake runs entirely in your browser — it reads only the sprite sectors and writes them back in place. Your ROM is never uploaded. The first bake auto-saves a pristine track03.bin.bak next to it; to undo, delete the edited file and rename the .bak back.
  4. Load the .gdi in flycast — or burn / GDEMU for real Dreamcast.
Serve the page over localhost (e.g. python -m http.server) — the file picker needs a secure context. Firefox/Safari lack File System Access → fall back to the server bake: python tools/skin_server.py. Painter needs the edit bundle once per char: python tools/export_editor_bundle.py PLxx.
Pixel-art bridge — edit in Aseprite / GIMP / Piskel
  1. Find which parts render a region (e.g. Cable's head) from the ASMTRACE set, then export them as indexed PNGs:
    python tools/part_png.py export PL17 198,205,210 ./edit
  2. Open the PL17_selNNN.png files in any editor. They're indexed (the character's 16 colors, index 0 = transparent) — keep indexed mode for exact results. Repaint freely.
  3. Bake with a skin.json that points at the edited PNGs:
    { "char":"PL17", "parts_png": { "198":"edit/PL17_sel198.png" } }
    python tools/bake_skin.py myskin.json
  4. Load the patched GDI — the edit renders on every frame that uses those sels, on flycast and real Dreamcast.
The bridge round-trips byte-exact (twiddle handled on the bake side) and goes through the offset-table rebuild, so edited art can be any size. Palette + parts_png can be combined in one skin.json.
